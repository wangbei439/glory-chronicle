## 二、今日实现功能

### 1. 角色移动系统
- **实现方式**：CharacterBody3D + 自定义 InputMap
- **按键绑定**：WASD / 方向键移动，Space 跳跃
- **关键配置**：
  - 创建自定义输入动作：`move_left`, `move_right`, `move_up`, `move_down`, `jump`
  - 不使用默认 `ui_*` 动作（避免与 UI 冲突）

### 2. 攻击系统
- **实现方式**：Area3D hitbox + `get_overlapping_bodies()`
- **按键**：J 键（`Input.is_key_pressed(KEY_J)`）
- **核心逻辑**：
  - hitbox 始终启用，攻击时检测重叠体
  - **必须 await 两个物理帧**才能获取正确的 overlapping bodies
  - 碰撞层：hitbox Layer 2 / Mask 1，Dummy Layer 1 / Mask 1

### 3. 闪避系统
- **按键**：K 键
- **参数**：0.4秒持续时间，速度15，1秒冷却
- **机制**：闪避期间设置 `hurtbox.set_deferred("monitoring", false)` 实现无敌帧

### 4. 敌人 AI 系统
- **3 状态机**：巡逻 (patrol) → 追击 (chase) → 攻击 (attack)
- **关键修复**：`find_player()` 必须先判断 attack_range 再判断 detect_range，否则 chase 永远覆盖 attack 状态

### 5. Boss 三阶段战斗
- **阶段转换**：
  - 100%~60%：正常状态
  - 60%~30%：橙色，速度+30%，伤害+50%，冷却-20%
  - 30%~0%：深红色，速度+60%，伤害+100%，冷却-40%
- **视觉反馈**：每阶段变色 + modulate 闪烁

### 6. 血条系统
- **实现方式**：MeshInstance3D + QuadMesh
- **结构**：hp_bg（黑色背景）+ hp_fill（红色填充）
- **材质**：Unshaded + Cast Shadow Off，确保任何光照下清晰可见

### 7. 伤害数字浮动
- **实现方式**：Label3D + Tween 动画
- **效果**：数字上浮 + 渐隐消失

### 8. 受击视觉反馈
- **实现方式**：`modulate = Color.WHITE`（闪白）→ 0.1秒后恢复
- **修复**：使用固定值 `Vector3(1.3, 1.3, 1.0)` 而非乘法 `* 1.3`，避免 scale 持续叠加

### 9. 装备系统
- **数据结构**：
  - `EquipmentData`（class_name，extends Resource）：部位枚举(Slot)、品质枚举(Quality)、基础属性
  - `.tres` 资源文件：iron_sword, copper_bow, leather_cap, leather_armor
  - `PlayerStats`（class_name）：HP/装备管理/属性计算/背包操作
- **注册技巧**：创建 `class_name` 后需 `Project → Reload Current Project` 才能在新建资源对话框中看到

### 10. 背包 UI
- **实现方式**：Control / Panel / VBoxContainer + 动态按钮
- **按键**：I 键打开/关闭

### 11. 装备掉落与拾取系统
- **掉落物**：Area3D + MeshInstance3D（发光球体）+ OmniLight3D（光效）+ CollisionShape3D（拾取范围）
- **动画**：sin() 浮动 + OmniLight3D.energy 脉冲
- **拾取**：走近自动拾取，调用 `PlayerStats.add_item()`
- **品质颜色**：白/绿/蓝/紫/橙 五级

## 三、踩坑记录（重要！）

| # | 问题 | 原因 | 解决方案 |
|---|------|------|----------|
| 1 | Sprite3D 没有 flip_h | Sprite3D 不像 Sprite2D 有 flip_h 属性 | 用 `$sprite.scale.x = -1` 实现翻转 |
| 2 | 角色看不到 | Sprite3D 没有贴图 | 手动添加精灵贴图 |
| 3 | get_overlapping_bodies() 返回空 | 物理引擎至少需要1帧更新 | `await get_tree().physics_frame` 两次 |
| 4 | body_entered 信号不触发 | hitbox 从 disabled→enabled 时，已在内部的物体不触发 | 改用 get_overlapping_bodies() 主动检测 |
| 5 | find_player() 覆盖 attack 状态 | 每帧先检测 detect_range 再检测 attack_range | 调整优先级：先 attack_range 再 detect_range |
| 6 | InputMap "attack" 动作不生效 | 原因不明，可能是 InputMap 配置问题 | 改用 `Input.is_key_pressed(KEY_J)` 直接检测 |
| 7 | 攻击 scale 持续叠加 | 用 `scale * 1.3` 每帧乘法 | 改用固定值 `Vector3(1.3, 1.3, 1.0)` |
| 8 | class_name 不出现在新建资源 | 需要重新加载项目 | Project → Reload Current Project |
| 9 | Dummy 脚本不生效 | 创建了 .gd 文件但没 Attach 到节点 | 在编辑器中选中节点 → Attach Script |
| 10 | GDScript 缩进报错 | Tab 和 Space 混用 | Edit → Convert Indentation → To Tabs |
| 11 | OmniLight3D.energy 赋值报错 | Godot 4.6.2 类型检查更严格 | 用 `$OmniLight3D.set("energy", value)` 绕过 |
